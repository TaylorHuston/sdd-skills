There should never be a question of what is actually implemented.

Epics describe durable behavior. Evidence points to the code and tests. Drift is a finding, not folklore.

01 / Process

A tool-neutral lifecycle

SDD defines the outcomes each stage must preserve. Teams can realize them with files, databases, work trackers, agents, or a combination.

The stage describes the method. The paired command shows how this package maps it into a local-first workflow.
  1. 01

    Explore

    Investigate a substantial product or technical question without prematurely turning it into implementation.

    Package/sdd-explore
  2. 02

    Capture intent

    Record the desired outcome now. Delay technical planning until the work is close enough to begin.

    Package/sdd-change --brief
  3. 03

    Plan just in time

    Refine Stories, Requirements, Scenarios, technical choices, and verification against current code.

    Package/sdd-change --plan
  4. 04

    Converge the experience

    For UI-bearing work, settle flow, responsive composition, states, accessibility, and visual direction.

    Package/sdd-design --plan
  5. 05

    Commit to a work scope

    Make the implementation-ready plan visible to the people and systems responsible for delivery.

    Packagesdd change promote
  6. 06

    Implement in verified slices

    Work Requirement by Requirement, prefer BDD/TDD, and keep Epic truth current as implementation changes.

    Package/sdd-apply
  7. 07

    Review independently

    Review the complete diff against its target and report technical readiness separately from manual acceptance.

    Package/sdd-review
  8. 08

    Reconcile and deliver

    Confirm the behavior map reflects what shipped, preserve unresolved gaps, and follow the team's acceptance and release policy.

    Package
    /sdd-pr /sdd-release

03 / Implementation

This package is one implementation of SDD

The methodology does not require this repository, Markdown, a CLI, a particular tracker, or a specific agent runtime.

Portable methodology

Story-Driven Development

  • Describe accepted behavior durably
  • Map behavior to implementation and evidence
  • Plan close enough to implementation to stay useful
  • Reconcile documentation as reality changes
  • Treat unexplained drift as a finding
Current reference implementation

Local-first SDD toolchain

  • Markdown Epics, Stories, Changes, and ADRs
  • Repository-local docs/ conventions
  • User topology in ~/.sdd/
  • Portable repository .sdd/ contracts
  • A deterministic CLI for structure and state
  • Agent skills for contextual judgment

Inside this implementation

Judgment where it matters. Automation where it helps.

Deterministic layer

SDD CLI

  • Install user skills and map private topology
  • Initialize portable repository contracts
  • Scaffold Changes and Epics
  • Validate artifact structure
  • Report status across repositories
  • Guard lifecycle transitions
Contextual layer

Agent skills

  • Refine outcomes and requirements
  • Compare technical paths
  • Route specialist guidance
  • Implement with evidence
  • Review the integrated diff
  • Audit broader codebase health

Optional component lifecycle

Standardize after use, not before it.

Shared catalogs can incubate reusable patterns without becoming mandatory application dependencies.

  1. 01CandidateRecord a reusable signal.
  2. 02PreviewExercise the important states.
  3. 03AdoptUse the project's ownership model.
  4. 04ValidateProve it in a real consumer.
  5. 05StandardizePromote only with evidence.

Application-specific components and deliberate divergences remain valid outcomes.

Install once, initialize each repository 
$ git clone https://github.com/TaylorHuston/sdd-skills.git
$ cd sdd-skills
$ npm install
$ npm link
$ sdd setup
$ cd /path/to/repository
$ sdd init
$ sdd doctor

Pre-1.0 distribution: install from the GitHub checkout for now. A standard npm registry install is targeted for the 1.0 release.

The North Star

Keep implemented behavior, code, and evidence in agreement.

No stale planning wiki. No game of telephone. No mystery about where to begin when something breaks.
